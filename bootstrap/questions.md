# Bootstrap Questions

Hỏi lần lượt từng nhóm. Không hỏi hết một lúc.
Dùng câu trả lời để chọn template và điền placeholder vào SKILL.md.

---

## Nhóm 1 — Project identity

**Q1.1** Tên project là gì?

**Q1.2** Mô tả ngắn gọn project làm gì? (1-2 câu)

**Q1.3** Project type:
- Web app (frontend / backend / fullstack)
- Data / automation / ML / ETL
- Embedded / firmware / hardware
- CLI tool / script
- Khác (mô tả)

---

## Nhóm 2 — Stack & environment

**Q2.1** Ngôn ngữ chính + framework/library chính?
> Ví dụ: Python + FastAPI, TypeScript + React, C++ + ROS2

**Q2.2** Môi trường chạy?
> Ví dụ: Linux server, Windows desktop, Raspberry Pi, browser

**Q2.3** Package manager / build tool?
> Ví dụ: pip + venv, npm, cargo, cmake

---

## Nhóm 3 — Data & I/O

**Q3.1** Nguồn dữ liệu đầu vào?
> Ví dụ: REST API, PostgreSQL, file CSV, serial port, camera

**Q3.2** Output / deliverable cuối cùng?
> Ví dụ: web dashboard, trained model, binary firmware, PDF report

---

## Nhóm 4 — Constraints

**Q4.1** Có constraint quan trọng nào không?
> Ví dụ: phải chạy offline, latency < 100ms, không dùng paid API, MIT license only

**Q4.2** Coding conventions muốn theo?
> Ví dụ: PEP8, Google style, tự đặt (mô tả ngắn)

**Q4.3** Test approach?
> Ví dụ: unit test bắt buộc, integration test thôi, không test (prototype)

---

## Nhóm 5 — Scope & priority

**Q5.1** MVP (Minimum Viable Product) của bạn là gì?
> Tính năng tối thiểu để gọi là "chạy được"

**Q5.2** Deadline hoặc milestone gần nhất?

**Q5.3** Có phần nào đã làm rồi, hay bắt đầu từ đầu hoàn toàn?

---

## Câu hỏi follow-up (hỏi thêm nếu câu trả lời chưa rõ)

- "Cụ thể hơn được không? Ví dụ..."
- "Có thể cho xem ví dụ về output mong muốn không?"
- "Constraint đó xuất phát từ đâu? (technical / business / personal)"
